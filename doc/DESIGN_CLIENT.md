# Generator (시물레이터) , Tester (API 테스터)

## Generator
* 도심지라 생각하고 속도는 70km 이하로 생각했다. 
* 앞서 설명한것 처럼 5 m/s = 18 km/h 이고  약 5 m 이동 할 때 위/경도는 0.00005 가 이동한다고 가정하였다.

### next 좌표의 생성
* 최초의 시작 점은 위도37 (3700000) 경도 127 (12700000) 으로 선언하였다.
* 현재 위치, 속도, 진행 방향 이 3개를 가지고 다음 좌표를 생성하였다.
* default:3 에 한번씩 데이터가 생성되며 -Dapp.client.time 값으로 이 주기는 변경 가능하다

### 고려사항
1. 네트워크 이슈 
   
   자동차의 네트워크 통신은 항상 불안정 하기 때문에 통신의 단절이나, 실패에 대해서 항상 염두에 둬야한다.
   * 통신이 단절되었을 경우 : 나중에 통신이 복구되었을 경우 과거의 데이터를 다시 전송해야 한다.
   * 데이터 생성 크기보다 데이터 전송 속도가 느릴 경우 : 속도가 느릴때에는 천천히, 빠를때에는 많이 보낼 수 있어야 한다.
   * 데이터의 생성 시간과, 서버에서의 수신 시간이 다르다 : 그렇기 때문에 데이터의 시간도 '위치 로의 생성 시간', '위치 로그가 서버에 저장된 시간' 2개가 다 필요하다. 
   
2. 잘못된 데이터의 생성
   
   네비게이션을 쓰면 잘못된 위치를 잡거나, 위치 자체를 잡지 못하는 경우가 많다.
   * Client (자동차) 리소스는 비싸고, 쉽게 업그레이드 할 수 없다. 
   * 즉시 필요한 데이터가 아니라면 가능한 최소한의 로직과 Validation 을 유지하고 서버에 위임하자
